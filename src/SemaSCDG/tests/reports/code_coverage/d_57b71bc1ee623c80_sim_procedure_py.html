19class SimProcedure: 

20 """ 

21 A SimProcedure is a wonderful object which describes a procedure to run on a state. 

22 

23 You may subclass SimProcedure and override ``run()``, replacing it with mutating ``self.state`` however you like, 

24 and then either returning a value or jumping away somehow. 

25 

26 A detailed discussion of programming SimProcedures may be found at https://docs.angr.io/extending-angr/simprocedures 

27 

28 :param arch: The architecture to use for this procedure 

29 

30 The following parameters are optional: 

31 

32 :param symbolic_return: Whether the procedure's return value should be stubbed into a 

33 single symbolic variable constratined to the real return value 

34 :param returns: Whether the procedure should return to its caller afterwards 

35 :param is_syscall: Whether this procedure is a syscall 

36 :param num_args: The number of arguments this procedure should extract 

37 :param display_name: The name to use when displaying this procedure 

38 :param library_name: The name of the library from which the function we're emulating comes 

39 :param cc: The SimCC to use for this procedure 

40 :param sim_kwargs: Additional keyword arguments to be passed to run() 

41 :param is_function: Whether this procedure emulates a function 

42 

43 The following class variables should be set if necessary when implementing a new SimProcedure: 

44 

45 :cvar NO_RET: Set this to true if control flow will never return from this function 

46 :cvar DYNAMIC_RET: Set this to true if whether the control flow returns from this function or not depends on 

47 the context (e.g., libc's error() call). Must implement dynamic_returns() method. 

48 :cvar ADDS_EXITS: Set this to true if you do any control flow other than returning 

49 :cvar IS_FUNCTION: Does this procedure simulate a function? True by default 

50 :cvar ARGS_MISMATCH: Does this procedure have a different list of arguments than what is provided in the 

51 function specification? This may happen when we manually extract arguments in the run() 

52 method of a SimProcedure. False by default. 

53 :cvar local_vars: If you use ``self.call()``, set this to a list of all the local variable 

54 names in your class. They will be restored on return. 

55 

56 The following instance variables are available when working with simprocedures from the inside or the outside: 

57 

58 :ivar project: The associated angr project 

59 :ivar arch: The associated architecture 

60 :ivar addr: The linear address at which the procedure is executing 

61 :ivar cc: The calling convention in use for engaging with the ABI 

62 :ivar canonical: The canonical version of this SimProcedure. Procedures are deepcopied for many reasons, 

63 including to be able to store state related to a specific run and to be able to hook 

64 continuations. 

65 :ivar kwargs: Any extra keyword arguments used to construct the procedure; will be passed to ``run`` 

66 :ivar display_name: See the eponymous parameter 

67 :ivar library_name: See the eponymous parameter 

68 :ivar abi: If this is a syscall simprocedure, which ABI are we using to map the syscall numbers? 

69 :ivar symbolic_return: See the eponymous parameter 

70 :ivar syscall_number: If this procedure is a syscall, the number will be populated here. 

71 :ivar returns: See eponymous parameter and NO_RET cvar 

72 :ivar is_syscall: See eponymous parameter 

73 :ivar is_function: See eponymous parameter and cvar 

74 :ivar is_stub: See eponymous parameter 

75 :ivar is_continuation: Whether this procedure is the original or a continuation resulting from ``self.call()`` 

76 :ivar continuations: A mapping from name to each known continuation 

77 :ivar run_func: The name of the function implementing the procedure. "run" by default, but different in 

78 continuations. 

79 :ivar num_args: The number of arguments to the procedure. If not provided in the parameter, extracted from 

80 the definition of ``self.run`` 

81 

82 The following instance variables are only used in a copy of the procedure that is actually executing on a state: 

83 

84 :ivar state: The SimState we should be mutating to perform the procedure 

85 :ivar successors: The SimSuccessors associated with the current step 

86 :ivar arguments: The function arguments, deserialized from the state 

87 :ivar arg_session: The ArgSession that was used to parse arguments out of the state, in case you need it for 

88 varargs 

89 :ivar use_state_arguments: 

90 Whether we're using arguments extracted from the state or manually provided 

91 :ivar ret_to: The current return address 

92 :ivar ret_expr: The computed return value 

93 :ivar call_ret_expr: The return value from having used ``self.call()`` 

94 :ivar inhibit_autoret: Whether we should avoid automatically adding an exit for returning once the run function 

95 ends 

96 :ivar arg_session: The ArgSession object that was used to extract the runtime argument values. Useful for if 

97 you want to extract variadic args. 

98

441 def call(self, addr, args, continue_at, cc=None, prototype=None, jumpkind='Ijk_Call'): 

442 """ 

443 Add an exit representing calling another function via pointer. 

444 

445 :param addr: The address of the function to call 

446 :param args: The list of arguments to call the function with 

447 :param continue_at: Later, when the called function returns, execution of the current 

448 procedure will continue in the named method. 

449 :param cc: Optional: use this calling convention for calling the new function. 

450 Default is to use the current convention. 

451 :param prototype: Optional: The prototype to use for the call. Will default to all-ints. 

452 """ 

453 self.inhibit_autoret = True 

454 

455 if cc is None: 

456 cc = self.cc 

457 prototype = cc.guess_prototype(args, prototype) 

458 

459 call_state = self.state.copy() 

460 ret_addr = self.make_continuation(continue_at) 

461 saved_local_vars = list(zip(self.local_vars, map(lambda name: getattr(self, name), self.local_vars))) 

462 simcallstack_entry = (self.state.regs.sp if hasattr(self.state.regs, "sp") else None, 

463 self.arguments, 

464 saved_local_vars, 

465 self.state.regs.lr if self.state.arch.lr_offset is not None else None, 

466 ret_addr) 

467 cc.setup_callsite(call_state, ret_addr, args, prototype) 

468 call_state.callstack.top.procedure_data = simcallstack_entry 

469 

470 # TODO: Move this to setup_callsite? 

471 if isinstance(call_state.addr, SootAddressDescriptor): 

472 pass 

473 elif call_state.libc.ppc64_abiv == 'ppc64_1': 

474 call_state.regs.r2 = self.state.mem[addr + 8:].long.resolved 

475 addr = call_state.mem[addr:].long.resolved 

476 elif call_state.arch.name in ('MIPS32', 'MIPS64'): 

477 call_state.regs.t9 = addr

479 self._exit_action(call_state, addr)
