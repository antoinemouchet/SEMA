71 """ 

72 Should merge the state plugin with the provided others. This will be called by ``state.merge()`` after copying 

73 the target state, so this should mutate the current instance to merge with the others. 

74 

75 Note that when multiple instances of a single plugin object (for example, a file) are referenced in the state, 

76 it is important that merge only ever be called once. This should be solved by designating one of the plugin's referees as the "real owner", who should be the one to actually merge it. 

77 This technique doesn't work to resolve the similar issue that arises during copying because merging doesn't produce a new reference to insert. 

78 

79 There will be n ``others`` and n+1 merge conditions, since the first condition corresponds to self.
